# 德鲁克式问题诊断框架

## 概述

德鲁克式诊断框架将管理学之父彼得·德鲁克的核心思想转化为结构化的问题诊断方法。适用于组织效率低下、战略迷失、增长停滞、流程冗余等场景。

**核心价值**：不是给答案，而是教你怎么问问题。从"功能本质"出发，剥离症状找根因，避免头痛医头。

---

## 核心功能

1. **功能定义检查**：任何组织/部门/产品的"功能"是什么？（先问目的，再看结构）
2. **结果检验**：实际产出 vs 期望产出，差距在哪里？
3. **根本原因分析**：如果只改一个点，哪个点能产生最大杠杆？
4. **知识工作者效能自检**：个人层面的有效性提升
5. **战略优先级矩阵**：区分"必须做"和"应该停"

## 核心公式与算法

### 1. 德鲁克四步诊断法

**步骤**：
```
1. 功能定义（What is it for?）
   → 问：这个组织/部门/产品的"功能"是什么？
   → 剥离结构，看目的。例如：人力资源部的功能不是"招人"，是"获取并保留人才"

2. 结果检验（What does it produce?）
   → 问：它产出的"结果"是什么？
   → 关注外部交换价值，不是内部活动。例如：市场部的产出不是"写文章"，是"获客"

3. 结果差距（Gap Analysis）
   → 问：实际结果 vs 期望结果，差距有多大？
   → 区分"因"和"果"：产出低是原因还是结果？

4. 根本原因（Root Cause）
   → 问：如果只改一个点，哪个点能产生最大杠杆？
   → 证据是什么？（数据、用户反馈、一线观察）
```

**产出**：诊断报告（根因 + 3个可执行改进建议）

---

### 2. 知识工作者效能三问（每日/每周自检）

**德鲁克核心思想**：知识工作者的生产力取决于"贡献"而非"忙碌"

**三问公式**：
```
效能总分 = (贡献清晰度 × 0.5) + (优势聚焦度 × 0.3) + (学习反馈环 × 0.2)

其中：
- 贡献清晰度：我的工作如何对齐组织目标？（1-10分）
- 优势聚焦度：我是否在做自己最擅长的事？（1-10分）
- 学习反馈环：每次项目后是否记录"学到了什么"？（是/否 × 10）
```

**行动规则**：
- 效能总分 < 6：本周必须调整工作内容
- 贡献清晰度 < 5：需要与上级重新对齐目标
- 优势聚焦度 < 5：考虑将低优势工作外包/委托

### 3. 战略优先级矩阵（Do the Right Things）

**德鲁克金句**："效率是用正确的方式做事，效能是做正确的事。效率是'如何'，效能是'什么'"

**矩阵**：
```
          高价值
            ↑
    II      |      I
合作/外包  |    All-in
            |
-----------|----------→ 高优势
            |
    III     |     IV
  停止      |  检查是否过度优化
            ↓  低价值
```

**操作**：
1. 列出所有项目/业务线
2. 评估：价值（高/低） × 优势（高/低）
3. 决策：
   - I（高价值×高优势）：投入最多资源，All-in
   - II（高价值×低优势）：合作/外包，不要自己All-in
   - III（低价值×高优势）：警惕"过度优化"，是否在错的事情上太努力？
   - IV（低价值×低优势）：立即停止

---

### 4. 有效决策五步法

**德鲁克观点**：有效决策不是"灵光一现"，是系统流程

**五步**：
```
1. 界定问题（Problem Definition）
   → 是"症状"还是"问题"本身？
   → 例：销售额下降（症状）→ 问题：客户流失率上升 or 新客户获取不足？

2. 定义边界条件（Boundary Conditions）
   → "必须满足什么条件" vs "希望满足什么条件"？
   → 强制约束（如预算≤$100K） vs 弹性约束

3. 正确方案（Right Solution）先于"接受度"（Acceptance）
   → 先找"什么是对的"，再考虑"怎么让人接受"
   → 避免妥协导致方案失效

4. 化决策为行动（Convert Decision into Action）
   → 谁？做什么？何时完成？需要什么资源？
   → 没有行动的决策=没有决策

5. 反馈机制（Feedback）
   → 预设验证点：6个月后检查决策效果
   → 如果结果不符预期，回到步骤1
```

## 使用示例

### 示例1：公司利润率下降

**四步诊断**：

1. **功能定义**：财务部的功能是"保证公司健康现金流"，不是"月底出报表"
2. **结果检验**：
   - 期望：利润率 ≥ 15%
   - 实际：利润率 8%（差距7%）
   - 产出：现金流健康度（评分：3/10，严重不合格）
3. **结果差距**：利润率低是"原因"还是"结果"？→ 是"结果"，真正问题是"成本结构不合理"
4. **根本原因**：
   - 假设1：原材料涨价 → 验证：采购价 vs 市场价，涨幅12% vs 市场涨幅5% → 部分原因
   - 假设2：生产效率低 → 验证：人均产出 vs 同行，低30% → **主要根因**
   - 假设3：定价过低 → 验证：产品定价 vs 竞品，低8% → 次要原因

**杠杆点**：提升生产效率（假设2），预计可提升利润率5-6个百分点

**改进建议**：
1. 立即启动精益生产项目（6周内完成）
2. 调整定价策略（分产品线，高价值产品提价10%）
3. 采购集中谈判（降低原材料成本3%）

### 示例2：个人效能诊断

**效能三问自检**（本周）：
- 贡献清晰度：我参与3个项目，只有1个明确对齐公司OKR → 评分4/10
- 优势聚焦度：我擅长数据分析，但花了60%时间在会议协调 → 评分3/10
- 学习反馈环：项目A学到了SQL优化，写了blog；项目B无记录 → 部分完成

**总分** = (4×0.5)+(3×0.3)+(5×0.2)=2+0.9+1=3.9 → **严重低于6，必须调整**

**行动**：
1. 与经理对齐，将60%会议协调转给另一位同事
2. 本月聚焦数据项目，申请退出2个低价值会议
3. 建立"项目复盘"模板，强制每次项目后填写

### 示例3：是否该砍掉亏损产品线

**战略优先级矩阵**：

产品线A：
- 价值：年营收$5M，利润率-5%（亏损）
- 优势：技术积累深，竞争对手难复制
- 位置：低价值 × 高优势 → IV区："检查是否过度优化"

产品线B：
- 价值：年营收$1M，利润率+20%（赚钱）
- 优势：技术一般，易被替代
- 位置：低价值 × 低优势 → III区："立即停止"

**决策**：
- 产品线A：不砍，但必须找到"如何变高价值"的路径（如升级、捆绑）
- 产品线B：剥离/出售，释放资源给A和核心业务

5. **变革管理**：公司为什么抵制变革？功能定义错误（旧功能 vs 新需求）

## 5个常见错误

1. **混淆症状与问题**：销售额下降（症状）→ 问题可能是产品过时、渠道失效、价格过高
2. **忽略边界条件**："增长"不能作为战略，必须有约束（如"有机增长""利润率≥10%"）
3. **追求完美方案**：德鲁克说"有效决策满足条件即可，不追求最优"
4. **只决策不行动**：很多公司做完战略规划就束之高阁，缺少"谁在什么时间完成什么"
5. **个人效能只关注时间管理**：德鲁克强调，知识工作者的核心不是"忙"，而是"贡献"
