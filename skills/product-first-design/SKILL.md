# 产品主义设计系统 v1.0

## 概述

产品主义设计系统（Product-First Design System）是一种将用户体验置于一切决策之上的产品开发方法论。灵感来源于史蒂夫·乔布斯（Steve Jobs）的产品哲学，强调闭环控制、极致体验、极简主义。

**核心理念**：
> "Real artists ship." — Steve Jobs

产品成功不是技术的堆砌，而是**用户体验的完整交付**。

---

## 功能

本 Skill 帮助产品团队：
- ✅ 建立以用户为中心的产品决策框架
- ✅ 学习苹果式的闭环生态设计
- ✅ 掌握"删减"而非"增加"的极简方法
- ✅ 制定产品路线图的优先级标准
- ✅ 构建跨部门的产品协作流程

## 使用方法

### 输入参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `action` | string | 是 | `evaluate-product` 或 `design-roadmap` 或 `prioritize-features` |
| `product-name` | string | 是 | 产品名称 |
| `product-type` | string | 是 | 产品类型（hardware/software/service） |
| `user-journey` | array | 否 | 用户旅程步骤（每个步骤的体验评分 1-10） |
| `features` | array | 否 | 功能列表，每个功能包含 name, effort (1-10), impact (1-10) |
| `competitors` | array | 否 | 竞争对手列表，用于对比分析 |
| `ecosystem` | boolean | 否 | 是否构建生态闭环（默认 false） |

### 输出

JSON 格式的产品评估报告或设计建议。

## 动作列表

### 1. evaluate-product

评估现有产品的"产品主义"程度，给出改进建议。

**示例输入**：
```json
{
  "action": "evaluate-product",
  "product-name": "智能音箱",
  "product-type": "hardware+software",
  "user-journey": [
    {"step": "购买", "score": 7},
    {"step": "开箱", "score": 8},
    {"step": "联网", "score": 6},
    {"step": "使用语音助手", "score": 9}
  ],
  "ecosystem": true
}
```

**示例输出**：
```json
{
  "product": "智能音箱",
  "productism-score": 78,
  "strengths": ["用户旅程流畅", "硬件质感好", "生态整合"],
  "weaknesses": ["设置步骤多", "隐私提示不足"],
  "recommendations": [
    "简化联网流程（目标：从6步减至3步）",
    "增加隐私控制透明度",
    "强化与智能家居设备的无缝连接"
  ],
  "apple-level": "Level 2/5（还有很大提升空间）"
}
```

### 2. design-roadmap

根据产品主义原则，制定产品路线图优先级。

**示例输入**：
```json
{
  "action": "design-roadmap",
  "product-name": "智能手表",
  "features": [
    {"name": "心率监测", "effort": 3, "impact": 8},
    {"name": "ecg 心电图", "effort": 5, "impact": 7},
    {"name": "睡眠追踪", "effort": 2, "impact": 6},
    {"name": "蜂窝网络", "effort": 8, "impact": 5},
    {"name": "双击支付", "effort": 4, "impact": 9}
  ],
  "budget-quarters": 4
}
```

**输出**：优先级排序的路线图（每个季度交付的功能）

---

### 3. prioritize-features

使用"删减法"对功能进行优先级排序。

**规则**：
- 总分 = impact - effort（简单版）
- 产品主义加成：如果功能直接提升用户体验，score × 1.5
- 最多保留 20% 高成本功能（effort ≥ 7）

**输出**：
- Must-have（Q1）
- Nice-to-have（Q2+）
- Kill-list（考虑删除）

## 原理

### 产品主义黄金公式

```
产品价值 = (核心功能 × 用户体验) - 复杂度
```

**其中**：
- **核心功能**：解决用户最痛的点（1-2个）
- **用户体验**：每一步的感受（购买、开箱、使用、售后）
- **复杂度**：学习成本 + 设置步骤 + 故障率

**目标**：最大化分子，最小化分母。

---

### 极简决策公式

```
每增加一个功能：
  if (effort > impact * 0.7) → 拒绝
  else → 接受，但必须删除两个旧功能
```

## 案例库（10个真实案例）

### 案例 1: iPhone 2007
- **核心功能**：触摸屏 + 互联网
- **用户体验**：开箱即用，无需说明书
- **复杂度**：极低（相比当时的黑莓、Palm）
- **结果**：颠覆手机行业

### 案例 2: iPod + iTunes
- **核心功能**：1000首歌装口袋
- **用户体验**：一键同步，iTunes Store 便捷购买
- **复杂度**：相比 Winamp 极简
- **结果**：音乐产业革命

### 案例 3: MacBook Air
- **核心功能**：轻薄 + 全功能笔记本
- **用户体验**：从信封中抽出，瞬间吸引
- **复杂度**：去掉了光驱、USB-A、以太网，但用体验补偿
- **结果**：定义了超极本

## 附录: 苹果产品主义检查清单

- [ ] 用户旅程是否 ≤ 5 步完成核心任务？
- [ ] 开箱体验是否"Wow"？
- [ ] 是否控制了完整的软硬件栈？
- [ ] 是否有"现实扭曲力场"级别的愿景？
- [ ] 是否在持续做减法而非加法？
- [ ] 品牌是否成为信仰？
- [ ] 生态锁定期望值是否 > 80%？
- [ ] 是否每个细节都经过千锤百炼？

**得分 ≥ 7/8 → 苹果级产品主义**
