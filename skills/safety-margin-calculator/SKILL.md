# 安全边际计算器

## 📋 概述

**安全边际计算器**是一个决策评估工具，源自沃伦·巴菲特的价值投资哲学。它的核心思想是：在做出任何重要决策前，先计算"内在价值"与"市场价格/代价"之间的差距，只有当这个差距达到一定安全阈值时才行动。

> "价格是你付出的，价值是你得到的。" —— 本杰明·格雷厄姆

## 🎯 核心功能

### 1. 多场景价值评估
- **投资决策**：股票、房产、基金等的内在价值 vs 市场价格
- **人才招聘**：候选人能力价值 vs 薪资成本
- **采购决策**：供应商质量 vs 报价
- **商业合作**：合作伙伴的长期价值 vs 资源投入
- **个人选择**：教育、健康、时间投入的回报评估

### 2. 安全边际阈值
- **保守型**：安全边际 ≥ 30%（适合高风险领域）
- **均衡型**：安全边际 ≥ 20%（默认）
- **积极型**：安全边际 ≥ 10%（适合机会成本低的场景）

### 3. 三重验证机制
1. **定量计算**：财务数据、历史业绩、市场对比
2. **定性评估**：团队质量、商业模式、护城河
3. **时间衰减**：安全边际随时间贬值（等待成本）

## 🧮 核心公式

```javascript
// 安全边际计算
margin_of_safety = (intrinsic_value - market_price) / market_price

// 综合得分（考虑概率）
expected_score = (margin_of_safety × success_probability) - (loss_if_failure × failure_probability)

// 最终决策
if (margin_of_safety ≥ threshold) && (expected_score > 0) {
  决策 = "执行";
} else {
  决策 = "放弃/等待";
}
```

**参数解释：**
- `intrinsic_value`：通过DCF、可比公司、资产重置等方法估算的真实价值
- `market_price`：当前市场价格或代价（价格、成本、机会成本）
- `success_probability`：基于历史数据、团队、市场的成功概率（0-1）
- `loss_if_failure`：失败时的损失幅度（相对比例）
- `threshold`：安全边际阈值（0.1-0.3）

## 📊 使用示例

### 示例 1：股票投资

```yaml
场景：评估某科技公司股票
当前股价：$100
估算内在价值：$150（基于DCF、PE对比、增长预期）
安全边际：(150 - 100) / 100 = 50%
成功概率：70%（基于行业地位、现金流）
失败损失：-40%（股价可能跌到$60）
预期得分：(0.5 × 0.7) - (0.4 × 0.3) = 0.35 - 0.12 = 0.23 > 0
决策：✅ 买入（安全边际50%远超20%阈值）
```

### 示例 2：人才招聘

```yaml
场景：招聘高级工程师
候选人市场价：$150k/年
估算价值：$200k/年（基于技能、经验、团队贡献）
安全边际：(200 - 150) / 150 = 33.3%
成功概率：80%（技术匹配、文化契合）
失败损失：-50%（不合适需重新招聘，成本$75k）
预期得分：(0.333 × 0.8) - (0.5 × 0.2) = 0.266 - 0.1 = 0.166 > 0
决策：✅ 录用（安全边际33.3% > 20%阈值）
```

### 示例 3：供应商选择

```yaml
场景：选择原材料供应商
供应商A报价：$100/吨（质量评级8/10）
Supplier B报价：$120/吨（质量评级9/10）
估算价值（基于质量、可靠性、长期合作）：
  A: $110（安全边际 10%）
  B: $150（安全边际 25%）
决策：✅ 选择B（质量更高，安全边际更大）
```

## 📈 评估框架

### Step 1：估算内在价值

**方法选择（根据场景）：**
- **DCF模型**：现金流折现（成熟企业）
- **可比公司**：PE/PB/PS对比（上市公司）
- **资产重置**：重制成本法（资产密集型）
- **收入倍数**：ARR × 倍数（SaaS/订阅）
- **专家评估**：领域专家打分（创新项目）

**三因素调整：**
```
最终价值 = 基础价值 × 管理系数 × 护城河系数 × 行业系数
```

### Step 2：确定市场价格

- 市场公开价格（股票、房产）
- 竞品报价（采购、供应商）
- 机会成本（时间、资金的其他用途）
- 隐性成本（整合、培训、风险溢价）

### Step 3：计算安全边际

```
margin_of_safety = (intrinsic_value - market_price) / market_price
```

**解读：**
- `> 30%`：极度安全，重仓/全力投入
- `20-30%`：安全，正常执行
- `10-20%`：边缘，需额外验证
- `< 10%`：危险，放弃或重新谈判

### Step 4：概率加权

```
expected_score = (margin × success_prob) - (loss × failure_prob)
```

**成功概率评估维度：**
- 历史记录（过去3年成功率）
- 团队质量（背景、经验、执行力）
- 市场环境（增长、竞争、监管）
- 技术/产品优势（护城河强度）

### Step 5：时间衰减

```
adjusted_margin = margin × (1 - time_decay_rate × months_waiting)
```

**原因：** 等待机会有成本（资金占用、竞争对手行动、市场变化）

## 🎓 应用场景

### 1. 投资组合管理
- 每笔投资前强制计算安全边际
- 安全边际 < 20% 时停止买入
- 安全边际 < 10% 时考虑卖出

### 2. 创业决策
- 项目投入前评估：预期价值 vs 资源消耗
- 融资时：估值 vs 实际价值（避免泡沫）
- 招聘：人才价值 vs 薪资成本

### 3. 企业采购
- 供应商选择：质量调整后价格 vs 市场平均
- 设备采购：生产效率提升 vs 购买成本
- 外包决策：自建成本 vs 外包价格

### 4. 个人发展
- 教育投资：技能提升带来的收入增长 vs 学费+时间
- 健康支出：预防性医疗的长期价值 vs 费用
- 时间分配：高价值活动 vs 低价值活动

### 5. 并购谈判
- 目标公司真实价值 vs 报价
- 协同效应的合理估值
- 整合成本的估算

## ⚠️ 常见错误

1. **过度乐观估值**：内在价值故意高估 → 安全边际虚高
   - 解决：使用保守假设，找第三方验证

2. **忽略流动性**：安全边际高但难以变现
   - 解决：流动性折扣（非流动资产 × 0.7-0.9）

3. **忽视时间价值**：长期投资未折现
   - 解决：DCF必须折现，机会成本计入

4. **单一数据源**：只依赖一种估值方法
   - 解决：至少3种方法交叉验证，取中位数

5. **忽视护城河**：高安全边际但无竞争壁垒
   - 解决：护城河系数 < 0.8 时额外折扣
