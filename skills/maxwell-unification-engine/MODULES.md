# 模块详细指南

## 模块 1: 现象统一法 (Phenomenological Unification)

### 核心思想
**"所有表面不同的现象，可能只是同一底层方程的不同约束条件。"**

麦克斯韦看到静电、静磁、电磁感应三个独立现象，发现它们共享电场 E 和磁场 B，于是构造了麦克斯韦方程组，统一了所有电磁现象。

### 为什么需要统一
- **维护成本**: 碎片化导致每个模块都要单独维护、演化和修复
- **认知负荷**: 学习 N 个独立系统 → 理解 1 个统一系统
- **创新能力受限**: 无法跨领域迁移发现
- **效率低下**: 重复解决本质相同的问题

### 核心算法步骤

#### Step 1: 绘制现象地图

```yaml
现象 P = [p₁, p₂, ..., pₙ]
每个 pᵢ 的变量集 Vᵢ = {x, y, z, ...}
目标: find V_intersect = ⋂ Vᵢ
```

**工具**: MindMap 或 Excel，列出所有现象及其变量

**案例**: 电商系统
```
p₁: 订单 → V₁ = {user_id, order_id, amount, status, items}
p₂: 支付 → V₂ = {user_id, transaction_id, amount, status, method}
p₃: 仓储 → V₃ = {sku, warehouse_id, quantity, location}
p₄: 物流 → V₄ = {order_id, tracking_no, carrier, address}
...
V_intersect = {user_id}  # 只有1个共享变量 → 碎片化严重!
```

#### Step 2: 构建核心方程

```
核心方程 E_core: f(V_intersect, 副变量₁, 副变量₂, ...) = 结果
```

**原则**:
- 使用最小变量 (Occam's Razor)
- 方程要对称/简洁 (数学美感)
- 约束条件清晰 (每个现象 = E_core + 约束)

**案例**: 统一电商核心模型
```
核心实体: User, Item, Transaction
核心关系:
  User --[购买]--> Item (通过 Transaction)
  Transaction.amount = Σ(Item.price × quantity)
  Transaction.status ∈ {pending, paid, shipped, delivered, cancelled}

各现象作为 Transaction 的视图:
  订单服务: all Transaction where type='order'
  支付服务: all Transaction where type='payment' & status='paid'
  物流服务: all Transaction where type='shipment'
  仓储: Item 库存 (独立实体, 但通过 SKU 关联)

统一度 U 从 0.2 → 0.8
```

#### Step 3: 验证分解

```
对每个原现象 pᵢ:
  添加约束 Cᵢ 到 E_core → 应还原 pᵢ 的完整行为
  测试: 在约束下, E_core 是否能生成 pᵢ 的所有记录?
```

**麦克斯韦验证**:
- 静电场: ∂B/∂t=0 → 方程1 → 库仑定律 ✓
- 恒定电流: ∂E/∂t=0 → 方程4 → 安培定律 ✓
- 变化场: 完整方程 → 电磁波预言 ✓

#### Step 4: 度量和优化

```
统一度 U = |V_intersect| / |⋃ Vᵢ|
目标: U > 0.7

方程质量 Q = C × K × P_new
  C = 简洁度 = 1/(项数×参数数)  # 越高越好
  K = 覆盖率 = |还原的现象数| / |总现象数|
  P_new = 新预言数 (已验证)
```

## 模块 2: 微分方程预测法 (Differential Forecasting)

### 核心思想
**"知道某个量如何变化 (导数)，比知道该量的绝对值更有预测力。"**

麦克斯韦不需要测量整个空间的 E,B，只需知道它们如何相互生成 (∇×B = μ₀ε₀∂E/∂t)，就能预言电磁波的存在和速度。

### 为什么用微分方程

- **数据效率**: 导数基于局部信息，无需全局采样
- **外推能力**: 模型外推到未观测区域
- **机制解释**: f(X) 本身是因果解释，非黑箱拟合
- **控制能力**: 知道 dX/dt 就能干预 (改变输入)

### 核心算法

#### Step 1: 定义状态变量

```
X(t) = [x₁(t), x₂(t), ..., xₖ(t)]  # 需要预测的 K 个量
```

**示例**:
- 商业: X = [用户数 U, 收入 R, 成本 C]
- 物理: X = [位置 r, 速度 v, 加速度 a]
- 生态: X = [猎物 P, 捕食者 Q]

#### Step 2: 建模变化率

```
dX/dt = f(X, t, θ)  # θ 是参数向量
```

**常见 f 形式**:
1. **线性**: dX/dt = A·X + b (可解析解)
2. **Logistic增长**: dU/dt = αU(1-U/K) (S型曲线)
3. **Lotka-Volterra**: dP/dt = αP - βPQ, dQ/dt = δPQ - γQ (振荡)
4. **自定义**: 基于理论推导

**参数估计**: 用历史数据拟合 (最小二乘, MLE)

#### Step 3: 求解与预测

```
解析解: X(t) = f⁻¹(...)  # 如果可能
数值解: X(t+Δt) = X(t) + Δt·f(X(t), t, θ)
```

**工具**: `scipy.integrate.odeint`, `solve_ivp`

#### Step 4: 识别平衡点与稳定性

```
平衡点 X*: f(X*) = 0
稳定性: 计算 Jacobian J = ∂f/∂X at X*
  特征值 λ: Re(λ)<0 稳定, Re(λ)>0 不稳定
```

**商业预测**: 预计系统会收敛到哪个 X* → 长期目标

#### Step 5: 验证

```
短期准确率: |X_pred(Δt) - X_true(Δt)| / |X_true|
长期趋势: sign(dX/dt) vs sign(真实变化)
参数可解释性: θ 是否有业务含义? (如 α = 增长率)
```

## 模块 3: 边界条件驱动创新 (Boundary-Led Innovation)

### 核心思想
**"系统的边界 (极限情况、切换点、奇异点) 是假设最脆弱的地方，也是创新杠杆最大的地方。"**

麦克斯韦发现安培定律 ∇×B = μ₀J 在电容器边界 (J=0 但 B≠0) 失效，添加位移电流项解决了。

### 边界类型

| 类型 | 数学表达 | 商业类比 |
|------|----------|----------|
| 几何边界 | r→0 或 r→∞ | 成本→0 或 规模→∞ |
| 时间边界 | t=0, t→∞ | 产品生命周期起点/终点 |
| 参数边界 | θ→0 或 θ→∞ | 极端用户需求 |
| 切换边界 | 约束条件改变 | 业务模式切换点 |
| 奇异点 | 分母=0, 导数无穷 | 系统崩溃点、突破点 |

### 创新四步法

#### Step 1: 描述系统方程

```
S: 系统行为 = f(变量, 参数, 约束)
示例: 利润 = (价格-成本)×销量
```

#### Step 2: 识别边界

```
边界 B = {b₁, b₂, ...} 其中 b 满足:
  - g(变量,参数)=0  (等式约束切换)
  - 变量→极值 (极限情况)
  - 导数→∞  (奇点)
```

**案例**: SaaS 定价
```
方程: 收入 = n×p (n=客户数, p=单价)
约束1: 小客户要求 p→0
约束2: 大客户要求 p ←→ ∞?  No, 要求 p 增长慢于 n
边界: n→0 时 p 必须有限;, n→∞ 时 p/n → 0
缺失项: 传统公式无网络效应 → 边界条件不满足

添加网络效应 k(n) = 1 + γ×(n-1)
新公式: 收入 = n×p×k(n)
边界: n→0 → k→1 → p可以 >0 (小客户补贴)
      n→∞ → k→∞ (但适度, γ<<1) → 大客户愿意付更多
```

#### Step 3: 检查完备性

```
在边界 b ∈ B 上:
  是否 f(..., b) 满足所有约束?
  是否产生矛盾 (如 ∞ - ∞)?
  是否丢失某些项?
```

**判断标准**: 若边界条件被违反，原方程不完整 → 需要添加 Δ 项

#### Step 4: 添加 Δ 并重构

```
新方程: f'(系统) = f(系统) + Δ(边界变量)
目标: Δ 能让边界条件满足, 且简化内部方程
```

**麦克斯韦**: Δ = μ₀ε₀∂E/∂t  
**商业**: Δ = 网络效应 k(n) × 基础价格

## 模块 4: 验证预言驱动研究 (Prediction-Driven Research)

### 核心思想
**"理论的价值 = 预言的复杂度 × 验证的严谨度"**

麦克斯韦方程组预言: 电磁波速度 = 1/√(μ₀ε₀) ≈ 3×10⁸ m/s → 与已知光速一致 → 预言"光就是电磁波" → Herschel, Hertz 验证。

### 研究循环

```
[理论 T] → [预言 P_new] → [实验 E] → [结果 R]
         ↖_______________反馈↙
```

**关键**: P_new 必须满足:
1. **新颖**: 不是构建 T 时已使用的数据
2. **可检验**: E 可行且成本合理
3. **明确**: 量化结果 (数值、范围、概率)

### 预言质量指标

```
1. 意外度 Surprise = P(data|null_model) / P(data|T)  # >1 才算意外
2. 可检验性 Testability = 1 / (实验成本 + 时间)  # 越高越好
3. 复杂度 Complexity = (新参数数) / (方程数)  # 越新越好, 但不能冗余

综合预言分 Score = w₁·Surprise + w₂·Testability + w₃·Complexity
```

### 实验设计

#### 类型 A: 决定性实验
```
预言 T₁: "如果 A 则 B", 预言 T₂: "如果 A 则非 B"
实验 E 结果要么支持 T₁ 要么支持 T₂
→ 快速淘汰错误理论
```

**案例**: 麦克斯韦预言电磁波 → Hertz 产生并测量 → 证实

#### 类型 B: 渐进验证
```
多个小型实验 E₁, E₂, ..., Eₙ
每个支持 T 的一个方面
累积置信度 = Π P(Eᵢ|T)
```

**案例**: 量子力学的多次实验验证

#### 类型 C: 有罪推定
```
预言: "如果 T 正确, 我们不应观察到 X"
实验: 寻找 X
如果发现 X → T 被否定
```

**案例**: 以太漂移实验 (Michelson-Morley) → 否定以太 → 相对论

---

### 商业实验设计 (A/B测试视角)

```
假设 H₀: 干预 I 无效应 (Δ=0)
假设 H₁: I 有效应 (Δ≠0)

 prophecy: "如果我们增加发送时间从 10am → 11am, 打开率提升 ≥5%"

实验设计:
  - 控制组: n₁ 用户, 10am
  - 实验组: n₂ 用户, 11am
  - 随机分配, 双盲 (如可能)
  - 收集指标: open_rate₁, open_rate₂

 统计检验: Δ̂ = rate₂ - rate₁
  p_value = P(Δ̂ ≥ observed | H₀)  或  置信区间

 结论: 若 p<0.05 且 Δ̂>0.05 → 预言验证
```

### M1.1 示例答案
```
现象: 用户注册 (邮箱), 用户注册 (手机), 用户绑定微信
变量: 
  邮箱注册: {email, password, ip, time}
  手机注册: {phone, sms_code, ip, time}
  微信绑定: {wechat_openid, user_id, time}

V_intersect = {ip, time}  # 只有2个 → U = 2/(5+5+3)=0.25

统一 E_core: 身份事件 = {identity_type, identifier, user_id?, context}
约束:
  类型=邮箱 → identifier=email, 需 password 字段
  类型=手机 → identifier=phone, 需 sms_code 字段
  类型=微信 → identifier=wechat_openid, 需 已存在 user_id

用一个"身份事件表"替换3个表, U→0.75
```

### M2.1 示例答案
```python
from scipy.optimize import curve_fit
import numpy as np

def logistic(t, alpha, K, U0):
    return K / (1 + (K/U0 - 1) * np.exp(-alpha * t))

t = np.array([0, 3, 6])
U = np.array([10000, 25000, 60000])

popt, _ = curve_fit(logistic, t, U, p0=[0.1, 100000, 10000])
alpha, K, U0 = popt
print(f"α={alpha:.4f}, K={K:.0f}, U0={U0:.0f}")

# 预测 t=12
U12 = logistic(12, *popt)
print(f"12个月后用户数: {U12:.0f}")

# 平衡点就是 K, 稳定 (Logistic总是稳定)
```
