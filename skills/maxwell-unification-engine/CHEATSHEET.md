# 速查表 (Cheatsheet)

## 现象统一法 (M1)

### 核心公式
```
统一度 U = |V_intersect| / |⋃ Vᵢ|
质量 Q = C × K × P_new
  C = 1/(项数×参数数)  # 简洁度
  K = |还原的现象数|/|总现象数|  # 覆盖率
  P_new = 新预言数 (已验证)
```

### 判断标准
- U > 0.7: 高度统一 ✅
- U 0.4-0.7: 部分统一 ⚠️
- U < 0.4: 碎片化 ❌ 需要重构

### Step 快速记
1. **画地图**: 列出所有现象及其变量
2. **找交集**: V_intersect = ⋂ Vᵢ
3. **建核心**: E_core: f(V_intersect, ...)=结果
4. **验分解**: E_core+约束 → 还原各现象?
5. **度质量**: 计算 U, Q, 迭代优化

## 微分方程预测法 (M2)

### 常用模型

| 模型 | 方程 | 解 | 应用 |
|------|------|-----|------|
| 指数增长 | dX/dt = αX | X=X₀e^{αt} | 病毒传播、复利 |
| Logistic增长 | dX/dt = αX(1-X/K) | S型曲线 | 用户渗透、市场饱和 |
| 衰减 | dX/dt = -βX | X=X₀e^{-βt} | 放射性衰变、遗忘 |
| 振荡 | d²X/dt² + ω²X = 0 | X=A sin(ωt+φ) | 季节性、物理振动 |
| Lotka-Volterra | dP/dt=αP-βPQ, dQ/dt=δPQ-γQ | 周期性振荡 | 生态竞争、供需 |

### 关键概念
- **平衡点** X*: f(X*)=0
- **稳定性**: |df/dX at X*|<1 → 稳定; >1 → 不稳定
- **数值求解**: X(t+Δt)=X(t)+Δt·f(X(t))

### 预测质量
```
质量 = (短期准确率×0.3) + (长期趋势正确×0.5) + (参数可解释性×0.2)
```

## 边界条件驱动创新 (M3)

### 边界类型速查

| 边界 | 数学 | 商业 | 示例 |
|------|------|------|------|
| 几何边界 | r→0 或 r→∞ | 成本→0, 规模→∞ | 用户数 n→0 时价格 p→? |
| 时间边界 | t=0, t→∞ | 产品起点/终点 | 冷启动问题 (t=0) |
| 参数边界 | θ→0 或 θ→∞ | 极端需求 | SLO 99.999% 可用性 |
| 切换边界 | 约束改变 | 业务模式切换 | 免费→付费的临界点 |
| 奇异点 | 分母=0, 导数∞ | 系统崩溃点 | 网络效应临界规模 |

### 创新四步法
1. **写方程**: 系统行为 = f(变量,参数,约束)
2. **找边界**: 列出所有边界 B
3. **验完备**: 在边界 b∈B 上, f 是否满足所有约束?
4. **加Δ项**: 添加 Δ(边界变量) → 新方程

### 案例模板
```
原方程: Y = f(X)  (如 收入 = 用户数 × 单价)
边界: X→0 时 Y→? 应满足: ...
        X→∞ 时 Y→? 应满足: ...
问题: 边界条件被违反
Δ:   = k(X) × f(X)  或  + g(X)
新方程: Y' = f(X) + Δ
验证: 边界条件满足 ✓
```

## 验证预言驱动研究 (M4)

### 预言三要素
1. **新颖**: 不是拟合用的数据
2. **可检验**: 实验可行, 成本合理
3. **明确**: 量化 (数值/范围/概率)

### 预言质量
```
预言分 = w₁·Surprise + w₂·Testability + w₃·Complexity
  Surprise = P(data|null_model) / P(data|T)  # >1
  Testability = 1/(实验成本+时间)
  Complexity = (新参数)/(方程数)
```

### 实验类型

| 类型 | 特点 | 用途 | 案例 |
|------|------|------|------|
| 决定性实验 | 预言 T₁ vs T₂ 互斥 | 快速淘汰 | Hertz 验证电磁波 |
| 渐进验证 | 多个小型实验累积 | 复杂理论 | 量子力学测试 |
| 有罪推定 | "不应观察到 X" | 否定旧理论 | Michelson-Morley |

### A/B测试预言模板
```
假设 H₀: 干预 I 无效 (Δ=0)
预言: "I 导致指标 M 提升 Δ_target (≥5%)"
实验: 
  - 控制组: n₁ 用户, 基线
  - 实验组: n₂ 用户, 干预
  - 随机, 双盲
  - 收集 M₁, M₂
统计: Δ̂ = M₂-M₁, p-value, 置信区间
决策: p<0.05 且 Δ̂≥Δ_target → 预言验证
```

## 统一到麦克斯韦

### 电磁学四方程
```
∇·E = ρ/ε₀          # 电荷产生电场
∇·B = 0              # 无磁单极
∇×E = -∂B/∂t         # 变化磁场产生电场 (法拉第)
∇×B = μ₀J + μ₀ε₀∂E/∂t  # 电流+变化电场产生磁场 (安培-麦克斯韦)
```

**边界创新**: 电容器充电时 J=0 但 ∂E/∂t≠0 → 发现缺失 μ₀ε₀∂E/∂t (位移电流)

**预言**: 联立 → ∇²E = μ₀ε₀∂²E/∂t² → 波方程 → 速度 v=1/√(μ₀ε₀)≈3×10⁸ m/s  
**实验**: Hertz (1887) → 确认

## 商务场景公式速查

### 统一度计算
```python
def unification_score(variable_sets):
    """variable_sets = [{v1, v2, ...}, {...}, ...]"""
    union = set().union(*variable_sets)
    intersection = set.intersection(*variable_sets)
    return len(intersection) / len(union)
```

### Logistic 增长
```python
def logistic(t, alpha, K, U0):
    return K / (1 + (K/U0 - 1) * np.exp(-alpha * t))
```

### 微分方程数值解 (RK4)
```python
def rk4_step(f, X, t, dt):
    k1 = f(X, t)
    k2 = f(X + dt*k1/2, t + dt/2)
    k3 = f(X + dt*k2/2, t + dt/2)
    k4 = f(X + dt*k3, t + dt)
    return X + dt*(k1 + 2*k2 + 2*k3 + k4)/6
```

### 平衡点稳定性
```python
def stability(f, X_star):
    """f: Rⁿ→Rⁿ, X_star: 平衡点"""
    # 数值 Jacobian
    eps = 1e-6
    n = len(X_star)
    J = np.zeros((n,n))
    for i in range(n):
        dx = np.zeros(n); dx[i] = eps
        J[:,i] = (f(X_star+dx) - f(X_star-dx)) / (2*eps)
    eigvals = np.linalg.eigvals(J)
    return np.all(np.real(eigvals) < 0)  # 全负则稳定
```

## 决策树

```
问题: 系统碎片化?
  → 用 M1: 统一度 U < 0.7? → 重构核心方程

问题: 趋势难以预测?
  → 用 M2: 建模 dX/dt, 仿真预测

问题: 需求冲突 ("既要又要")?
  → 用 M3: 分析边界, 找创新Δ

问题: 研究/产品方向不确定?
  → 用 M4: 提出可验证预言, 实验决定

全部需要?
  → M1→M3→M2→M4 (产品经理路线)
  → M2→M4→M1→M3 (数据科学家路线)
```

## 常见陷阱

| 模块 | 陷阱 | 如何避免 |
|------|------|----------|
| M1 | 过度统一 (U 高但实用性低) | 确保还原后的系统性能不降 |
| M2 | 过拟合历史数据, 外推失效 | 留出验证集, 参数可解释 |
| M3 | 边界项Δ使方程更复杂 | Δ应简化内部, 增加总复杂度≤10% |
| M4 | 预言模糊 ("会提升") | 必须量化: Δ≥5%, 置信区间 |
