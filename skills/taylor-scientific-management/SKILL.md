## 1. 概述

本 Skill 提取了弗雷德里克·温斯洛·泰勒（Frederick Winslow Taylor，1856-1915）在20世纪初创立的**科学管理**（Scientific Management）核心方法论，形成一套**可复制的效率革命框架**，适用于任何需要**标准化、可重复、可测量**的任务优化。

泰勒的核心贡献:
- **时间-动作研究**：用秒表（和电影胶片）分析工人动作，剔除浪费，制定"最佳工作方法"
- **标准化**：工具、环境、流程三维标准化，新员工培训从3个月→2周
- **差异计件制**：工资与效率强绑定，完成标准产量工资+30-50%，未完成-20-30%
- **职能制**：计划部与执行部分离，管理成为科学而非经验
- **持续改进**：效率记录簿 + 知识库 + 微创新测试标准化流程

适用场景:
- 制造业（装配、加工、质检）
- 服务业（呼叫中心、快餐、快递分拣）
- 知识工作（开发、设计、写作的流程优化）
- 医疗流程（手术步骤、护理规范）
- 任何可重复、可测量的任务（年重复次数>1000次）

## 2. 核心框架

### 2.1 时间-动作研究（用秒表革命）

**问题**: 传统"老师傅带徒弟"效率靠个人天赋，如何系统化提升？

**泰勒做法**: 4步法

1. **选择典型任务**: 选重复性高、耗时长的任务（如"铲煤"、"装配螺丝"）
2. **动作分解**: 将任务拆为最小单元（如"弯腰-抓铲-转身-举臂-投送"）
3. **时间测量**: 用秒表测每个单元，30-50次样本，去除异常值
4. **最佳方法测试**: 尝试不同姿势、工具、顺序，找到总时间最短的组合
5. **标准时间设定**: `standard_time = (平均时间 + 必要休息 allowance × 1.1~1.3`

**公式**:
```
efficiency = (标准时间/实际当前时间) × (标准方法质量评分/采用方法质量评分)
```

**案例**: 伯利恒钢铁铲煤工
- 原方法: 每人12.5吨/天，工具重量不一
- 泰勒分析: 100+种铲煤动作，测试不同铲子大小
- 最佳铲子: 根据煤种选择，标准重量16-18磅
- 结果: 人均47吨/天，效率提升 275%

---

### 2.2 标准化三维体系

**工具标准化**:
- 每个工位固定工具清单（大小、型号、数量）
- 工具摆放位置标记（"tool shadow board"）
- 定期检查维护，损坏立即更换

**环境标准化**:
- 工作台高度、照明亮度、温度湿度、噪音控制
- 物料摆放位置（5S：整理、整顿、清扫、清洁、素养）
- 休息频率（每50分钟5分钟）

**流程标准化**:
- 每个任务有SOP（Standard Operating Procedure），图文并茂
- 新员工按SOP操作，2周内达到标准
- SOP每年至少review一次，更新

**公式**:
```
training_time_reduction = (标准化前培训时间/标准化后培训时间)
目标: training_time_reduction > 3.0
```

---

### 2.3 差异计件制（把工资与效率绑定）

**问题**: 工人缺乏动力学标准方法，老方法"够用就行"。

**泰勒方案**: 两档工资率

| **完成情况** | **计件工资率** | **效果** |
|--------------|----------------|----------|
| ≥ 标准产量 | 高倍率（+30-50%） | 奖励高效率 |
| < 标准产量 | 低倍率（-20-30%） | 惩罚低效率 |

**关键**: 标准产量需合理（经时间-动作研究确定，是"合理"而非"不可能"）

**公式**:
```
earnings = units_produced × piece_rate × (若达标: 1.3~1.5; 若未达标: 0.7~0.8)
```

**激励系数**: `incentive_factor = (高倍率-1) × (达标率) - (1-低倍率) × (未达标率)`

**实施条件**:
1. 标准产量必须科学制定（工人认可）
2. 必须有精确计量（自动计数）
3. 工资周期短（每周发放）
4. 配合培训，让每个工人都能达标

---

### 2.4 职能制（管理科学化）

**传统**: 工头既管计划又管执行，靠个人经验。

**泰勒**: **计划-执行分离**

- **计划层**（Planning Department）:
  - 制定标准、工时定额
  - 设计工作方法
  - 跟踪实际数据
  - 持续改进

- **执行层**（Workers）:
  - 按标准方法执行
  - 报告问题（但不自己决定方法）
  - 接受培训

**新增管理角色**:
- 职能工头（8-10个，各负责一项职能：速度检查、质量检查、设备维护、物料配送）
- 计划员（制定每日生产计划）
- 成本员（核算效率）

**结果**: 管理从"手艺"变成"可复制的科学"

---

### 2.5 持续改进与知识积累

**泰勒 believes**: 效率提升永无止境。

**机制**:
1. **效率记录簿**: 每个工位/机器记录实际产出、停滞时间、异常事件
2. **微创新提案**: 工人可建议改进，经测试后若有效，奖励+标准化
3. **定期 re-study**: 每2年重新 time-study，标准时间持续降低
4. **知识库**: 所有标准方法、工具设计、计件数据入库，新项目参考

**公式**:
```
continuous_improvement_rate = (新标准时间 - 旧标准时间) / 旧标准时间 / 年
目标: 每年降低 5-10%
```

## 3. 实施流程 (Step-by-Step)

### 阶段1: 诊断与选择任务（2-4周）
1.1 选择3-5个高耗时、高重复任务（年重复>1000次）
1.2 测量当前：人均产出、质量标准、成本
1.3 评估潜力：预估效率提升空间（通常30-100%）

### 阶段2: 时间-动作研究（4-6周）
2.1 动作分解：将任务分解为10-30个最小单元
2.2 样本采集：30-50次完整任务，秒表记录每个单元
2.3 数据分析：剔除异常（疲劳、设备故障），计算平均值
2.4 最佳方法实验：改变工具/姿势/顺序，重复测量，找最小值
2.5 设定标准时间：`standard_time = best_time × (1 + 休息allowance 10-30%)`
2.6 制定SOP：图文+视频，标注关键动作

### 阶段3: 标准化建设（2-4周）
3.1 工具标准化：设计专用工具，采购并分配到工位
3.2 环境标准化：工作台高度、照明、物料摆放（5S）
3.3 培训新员工：用新SOP培训5-10人，测量达标时间
4.4 更新SOP：根据培训反馈修正

### 阶段4: 差异计件制设计（1-2周）
4.1 设定标准产量：`standard_output = (可用工时 × 3600) / standard_time`
4.2 确定工资率：
   - 基础件工资（市场价）
   - 高倍率（1.3-1.5）应用于达标者
   - 低倍率（0.7-0.8）应用于未达标者
4.3 沟通：与工人解释公式，强调"多劳多得"
4.4 试运行2周，收集反馈，调整标准产量（太松/太紧都不行）

### 阶段5: 职能制组织重构（4-8周）
5.1 设立计划部：配备计划员、调度员、成本员
5.2 执行层只保留"操作工"
5.3 新增职能工头（Quality Inspector, Method Supervisor, Maintenance Foreman）
5.4 明确权责：计划部产标准，工人按标准执行，职能工头监督

### 阶段6: 知识库与持续改进（持续）
6.1 建立效率数据库：每个工位每日产出、停滞原因
6.2 月度分析会：识别瓶颈，启动新时间-动作研究
6.3 微创新奖励：工人提案采纳后奖励+标准化
6.4 每2年 re-study，更新标准时间

## 4. 应用示例

### 案例A: 呼叫中心话务员

**任务**: 接听客户电话，解决问题。

**泰勒式改造**:
1. **时间-动作研究**: 记录1000次通话，分解为：振铃等待、问候、身份验证、问题诊断、解决方案、结束语
2. **最佳方法**: 标准话术（问候语、验证问题、解决方案模板），减少思考时间
3. **标准时间**: AHT（Average Handle Time）从 8分钟 → 5分钟
4. **标准化**: 知识库（FAQ）、脚本、键盘快捷键布局
5. **差异计件制**: 计件工资按"处理量×质量评分"，达标者+40%
6. **职能制**: 质检员（抽检录音）、培训师（新员工）、调度员（排班）
7. **结果**: 人均处理量+60%，满意度保持（质量没降），成本降40%

---

### 案例B: 软件开发代码审查

**任务**: Code Review，找出bug和改进建议。

**传统**: 工程师凭经验审查，耗时2-4小时/PR，质量不稳。

**泰勒式**:
1. **时间-动作研究**: 分析100个PR，分解：跑测试、读代码、查逻辑、写评论、沟通
2. **最佳方法**: 检查清单（Checklist） + 自动化工具（静态分析、测试覆盖率）
3. **标准时间**: 从3小时 → 1小时（清单+工具覆盖80%常见问题）
4. **标准化**: PR模板、审查清单、自动化CI报告
5. **差异计件制**: 按"审查数量×问题发现率"计件，达标者+35%
6. **职能制**: 自动化系统（CI）、审查专家（复杂PR）、调度员（分配PR）
7. **结果**: Review 效率提升200%，bug漏检率下降50%

---

### 案例C: 快餐店汉堡制作

**任务**: 制作一个汉堡（面包+肉饼+蔬菜+包装）。

**泰勒原著案例**: 1910年已广泛应用。

**现代应用**:
1. **时间-动作**: 录像分析每个动作：拿面包、烤、肉饼翻面、加料、包装
2. **最佳方法**: 传送带速度匹配人速，双手分工（左手拿面包，右手加料）
3. **标准时间**: 30秒/个（从90秒）
4. **标准化**: 工具位置标记、物料预分装、温度控制
5. **差异计件制**: 每完成100个，标准组奖励+50%
6. **职能制**: 专人烤肉、专人加料、专人包装、专人质检
7. **结果**: 产能3倍，单店人力成本降25%

## 6. 使用限制与注意事项

⚠️ **关键前提**:
1. **任务可重复**: 每年重复>1000次，时间-动作研究才有意义
2. **可测量**: 必须有精确计时和计数手段
3. **管理权威**: 能推行"计划-执行分离"，工人必须服从标准
4. **文化接受**: 劳资双方接受"效率提升是共赢"（避免工会抵制）
5. **资本投入**: 标准化工具、计划部门、质检员需要前期投入

**不适合场景**:
- 创意工作（艺术、写作、研发）——过度标准化扼杀创新
- 高频变化的环境（需求/技术/政策月月变）——标准很快过时
- 强 labor union 环境（工会反对计件制、职能制）
- 小作坊式（<10人）—— overhead 不划算

## 7. 历史验证与局限性

### 成功:
- 伯利恒钢铁: 铲煤工效率+275%
-  Midvale Steel: 机械加工效率+100-300%
- 美国海军兵工厂: 炮弹生产效率+200%
- 全球制造业: 1910-1930年代效率革命的基础

### 局限与失败:
- **人性忽略**: 工人被当机器，拒绝"去技能化"，1911年国会听证会批"奴隶制"
- **创新压制**: 标准方法压制工人微创新，长期损害适应性
- **过度分解**: 任务太小，工人感到无意义，高流失率
- **计件制争议**: 标准产量常被"操纵"，导致纠纷
- **知识流失**: 工人不思考"为什么"，只做"怎么做"，企业知识沉淀不足

**关键教训**: 泰勒主义是效率的"第一阶段"，适用于**重复、稳定、可测量**的环境。在知识经济时代，需要与"授权+创新"结合（如丰田的"建议系统"），而非纯命令-控制。

### 8.2 差异计件制计算器
```python
def piece_rate_calculator(standard_output, base_rate, high_mult=1.4, low_mult=0.75):
    """
    standard_output: 日标准产量（件）
    base_rate: 基础件工资（元/件）
    high_mult: 高倍率（>1）
    low_mult: 低倍率（<1）
    """
    def earnings(actual):
        if actual >= standard_output:
            return actual * base_rate * high_mult
        else:
            return actual * base_rate * low_mult
    return earnings
```
