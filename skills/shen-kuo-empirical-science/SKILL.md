## 🔧 技能模块 1: 实证观察法 (Empirical Observation)

### 功能描述
系统化观察现象，记录原始数据，避免先入为主的偏见。沈括在《梦溪笔谈》中记录了数十个自然现象的精确观察，如磁针"常微偏东，不全南也"。

### 适用场景
- 产品用户行为分析
- 市场趋势研究
- 技术问题诊断
- 科研数据收集
- 竞争对手分析

### 核心算法/公式
**观察框架 = 现象 + 数据 + 时间 + 环境 + 重复**

```
观察记录模板 = {
  "现象": "描述观察到的现象",
  "数据": "量化指标（数字、比例、频率）",
  "时间": "观察时间点/段",
  "环境": "条件、背景、变量",
  "重复": "验证次数与一致性",
  "反证": "尝试推翻观察的尝试"
}
```

### 使用示例
```json
{
  "任务": "分析新功能使用率下降",
  "观察": {
    "现象": "A功能月活跃使用率从25%降至12%",
    "数据": {"使用人数": 2500→1200, "使用时长": "3.2min→1.8min"},
    "时间": "2025-12至2026-02",
    "环境": "同期发布B功能、春节假期、竞品C新版本",
    "重复": "3次数据抽样，趋势一致",
    "反证": "排查数据统计错误（无）、服务器问题（无）"
  },
  "结论": "B功能替代效应显著，需优化A功能或合并"
}
```

## 🔧 技能模块 2: 跨学科迁移法 (Cross-Disciplinary Transfer)

### 功能描述
将其他领域的原理、技术或模式迁移到当前问题。沈括将数学"隙积术"（长方体累加原理）用于计算买卖亏算，将地质侵蚀原理用于解释化石成因。

### 适用场景
- 技术创新与专利设计
- 商业模式创新
- 解决跨领域复杂问题
- 寻找蓝海市场机会
- 产品功能借鉴

### 核心算法/公式
**迁移强度 = 源领域原理 × 目标场景匹配度 ÷ 转化复杂度**

```
迁移步骤:
1. 抽象化源领域核心原理（去场景化）
2. 识别目标问题的本质结构
3. 匹配结构相似性（S ≥ 0.6）
4. 设计转化实验（3个变量以内）
5. 验证最小可行性（2周内）

匹配度评分表:
- 静态结构相似: 0.8
- 动态过程相似: 0.7
- 因果关系相似: 0.9
- 反馈机制相似: 0.6
- 约束条件相似: 0.5
```

### 使用示例
**案例：将"水循环"原理迁移到知识管理系统**

```python
# 源领域：自然水循环（蒸发→云→雨→河流→海洋）
# 核心原理：物质循环利用、能量驱动、多态转换

# 目标问题：企业知识流失，员工离职带走经验

# 迁移设计：
知识蒸发 = 员工隐性知识 → 文档/课程（显性化）
知识云 = 知识库/文档系统（存储与分发）
知识雨 = 培训/分享会（传播）
知识河流 = 工作流程/标准操作（应用）
知识海洋 = 组织知识资产（沉淀）

# 转化复杂度：低（文档系统已存在，需改变激励）
# 匹配度：0.7 + 0.6 + 0.7 = 0.67
# 迁移强度 = 0.67 ÷ 2 = 0.34（中等强度，值得尝试）
```

## 🔧 技能模块 3: 系统化实验法 (Systematic Experimentation)

### 功能描述
设计可重复、可控制的实验来验证假设。沈括通过多次观察验证磁偏角，通过不同地区化石对比验证地质理论。强调"可重复性"和"对照"。

### 适用场景
- A/B测试设计
- 技术可行性验证
- 市场假设检验
- 产品功能优化
- 工艺改进

### 核心算法/公式
**实验有效性 = 控制变量数 × 样本量 × 重复次数**

```
实验设计模板:
{
  "假设": "如果X，则Y（可证伪）",
  "变量": {
    "独立": ["X1", "X2"],
    "依赖": ["Y1", "Y2"],
    "控制": ["C1", "C2", "C3"]
  },
  "样本": "最小n=30（统计显著）",
  "重复": "3次独立实验，变异率<10%",
  "对照": "基准组 vs 实验组（唯一差变量）"
}

沈括原则:
1. 三现主义: 现场、现物、现实
2. 多场景验证: 至少3个不同场景
3. 时间维度: 短期+长期效果
4. 边界测试: 找到失效临界点
```

### 使用示例
**案例：验证新定价策略对转化率的影响**

```json
{
  "假设": "如果价格从¥299降至¥199，转化率提升>20%",
  "设计": {
    "独立变量": "价格（¥299 vs ¥199）",
    "依赖变量": ["转化率", "客单价", "复购率"],
    "控制变量": ["页面设计", "流量来源", "促销文案", "产品版本"],
    "样本": "每组的独立用户n≥5000（95%置信）",
    "实验周期": "2周（覆盖周末效应）",
    "重复": "3个不同产品线分别测试"
  },
  "决策规则": {
    "转化率提升≥20%": "全量发布",
    "转化率提升5-20%": "扩大测试至50%用户",
    "转化率<5%": "放弃策略，回归原价"
  }
}
```

### 商业价值评估
- **风险降低**: 减少错误决策70%
- **决策速度**: 2-4周获得可靠结论（替代3-6月猜测）

## 🔧 技能模块 4: 知识结构化法 (Knowledge Structuring)

### 功能描述
将零散知识系统化、结构化，形成可检索、可传承的知识资产。沈括将毕生观察与研究整理为《梦溪笔谈》26卷， compartmentalize into 609个条目，涵盖17个学科。

### 适用场景
- 个人知识管理（PKM）
- 企业知识库建设
- 培训体系开发
- 技术文档组织
- 研究项目管理

### 核心算法/公式
**知识价值密度 = 有效信息量 ÷ 获取成本 × 复用频率**

```
沈括式结构化 = 知识树 × 索引系统 × 案例库

结构模板:
{
  "元知识层": {
    "分类体系": "17个大类（天文、地理、物理...）",
    "索引机制": "主题索引+交叉引用+时间线",
    "评价标准": "可信度、时效性、实用性"
  },
  "知识节点": {
    "核心命题": "1-2句话概括",
    "证据链": "3+个独立证据",
    "反例": "至少1个反例或边界条件",
    "应用场景": "2-3个实践用例"
  },
  "知识连接": {
    "因果连接": "A导致B",
    "类比连接": "A类似B",
    "时间连接": "A先于B",
    "对立连接": "A反驳B"
  }
}

最小可行知识结构 (MVKS):
- 每个知识点必须有：定义 + 证据 + 应用
- 每个知识域必须有：3个核心命题 + 5个连接点
- 知识库必须支持：(name) → (definition, evidence, applications)
```
