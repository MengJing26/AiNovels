## 核心框架：孙子战略公式

**系统公式**：**必胜战略 = (知己知彼 × 先胜后战) + (以迂为直 × 避实击虚) + (修道保法 × 形神合一)**

### 1. 五事七计：战略环境扫描器

**核心洞察**: 胜负在开战前就已决定，通过系统比较双方基本面。

- **五事**（战略基本面）：
  1. **道**：政治/组织共识度（上下同欲）
  2. **天**：时机与外部环境（天时、趋势）
  3. **地**：市场与竞争格局（地利、位置）
  4. **将**：领导者能力（战略判断、执行）
  5. **法**：组织能力与流程（制度、效率）

- **七计**（量化比较）：
  1. 主孰有道？ (领导力 vs 共识)
  2. 将孰有能？ (CEO vs 团队能力)
  3. 天地孰得？ (时机 vs 趋势)
  4. 法令孰行？ (制度 vs 执行力)
  5. 兵众孰强？ (资源 vs 资金)
  6. 士卒孰练？ (团队 vs 培训)
  7. 赏罚孰明？ (激励 vs 考核)

**算法**：
```
score_A = (道×0.2 + 天×0.15 + 地×0.15 + 将×0.25 + 法×0.25) * 执行系数
score_B = 同上
胜利概率 = logistic(score_A - score_B)  # logistic函数映射到[0,1]

如果 胜利概率 > 0.7 → 可战
如果 0.4-0.7 → 谨慎评估
如果 <0.4 → 不战（等待或转型）
```

### 2. 先胜后战：非对称优势创造

**核心原则**: "胜兵先胜而后求战，败兵先战而后求胜"

- **先胜** = 创造不可战胜的条件
  - 等待时机（"激水之疾，至于漂石"）
  - 积累势能（"势如彍弩，节如发机"）
  - 确保资源充足（"先为不可胜，以待敌之可胜"）

- **应用**：
  - 创业前：验证PMF（Product-Market Fit），确保有10倍好产品
  - 市场竞争：避开红海，寻找蓝海或创造新品类
  - 融资：估值合理而非追高，保留充足跑道

### 3. 以迂为直：非线性战略思维

**核心洞察**: 最短路径往往不是直线，而是绕道创造更大收益。

- **原文**: "以迂为直，以患为利"
- **商业应用**：
  - 短期亏损换取长期份额（AWS早期定价）
  - 开源商业模式（Red Hat, Elastic）
  - 平台战略先补贴生态（Uber, Airbnb）
  - 并购整合而非内部研发（Meta收购Instagram）

- **决策框架**：
```
评估"迂"的价值：
  迂回路径的长期收益L = 终点价值 × (1 + 增长加速因子)^时间
  直接路径的短期收益S = 终点价值 - 迂回成本
  如果 L > 1.5×S → 选择迂回
```

### 4. 避实击虚：蓝海与利基市场

**核心原则**: 不与强者正面竞争，攻击其薄弱环节。

- **"实"**：对手优势区（技术壁垒、品牌心智、网络效应）
- **"虚"**：未被满足的客户需求、新兴市场、低满意度环节

- **识别虚的指标**：
  - NPS < 30（客户不满）
  - 市场增长率 > 20%但份额分散
  - 技术迭代快，老玩家反应慢
  - 监管变化带来新机会

- **案例应用**：小米早期避开高端，专注性价比；拼多多避开一二线，切入下沉市场

### 5. 形神合一：战略与执行一致

**核心**: 战略（形）与组织能力（神）必须匹配。

- **"形"**：战略定位、资源配置、攻守节奏
- **"神"**：企业文化、团队士气、领导力魅力

- **对齐检查**：
  - 资源分配是否匹配战略优先级？
  - KPI是否驱动正确行为？
  - 团队是否理解并相信战略？
  - 激励制度是否支持长期主义？

## 可学习技能模块

### Skill 24.1: 五事七计战略诊断器

**功能**: 系统评估企业/项目在竞争环境中的位置，量化胜率

**算法**:
```
输入: 道(0-1), 天(0-1), 地(0-1), 将(0-1), 法(0-1), 执行系数(0-1)
处理:
  我方得分 = (道*0.2+天*0.15+地*0.15+将*0.25+法*0.25) * 执行系数
  对手得分 = 同上（需调研或估算）
  差值 = 我方得分 - 对手得分
  胜率 = 1 / (1 + exp(-差值*5))  # logistic映射，灵敏度调节

输出:
  胜率 > 0.7 → "可战：优势明显，建议进攻"
  0.4-0.7 → "谨慎：需寻找非对称优势"
  <0.4 → "不战：等待或转型"

同时输出短板分析：
  - 指出五事中得分最低项（最大短板）
  - 给出改进建议（如"道"低→加强文化共识，"法"低→优化流程）
```

**示例**:
```
初创公司 vs 行业龙头
我方: 道0.8(团队共识高), 天0.7(趋势好), 地0.6(市场中等), 将0.9(创始人强), 法0.4(流程弱), 执行1.0
对手: 道0.6, 天0.5, 地0.8, 将0.7, 法0.9, 执行0.8
胜率计算: 我方得分0.68, 对手得分0.73 → 差值-0.05 → 胜率≈0.4
建议: 不正面竞争，"避实击虚"→寻找细分市场或差异化定位
```

### Skill 24.2: 先胜后战资源蓄能系统

**功能**: 判断当前是否达到"先胜"条件，否则继续蓄能

**核心公式**:
```
先胜指数 = 资源充足度 × 势能积累度 × 时机成熟度

其中:
  资源充足度 = 现金跑道(月) / 12 + 收入稳定性系数
  势能积累度 = 产品独特性分(0-1) × 客户口碑(0-1)
  时机成熟度 = 市场增长率 × 竞争强度倒数

阈值:
  指数 > 0.8 → 可战
  0.5-0.8 → 继续蓄能（建议：产品迭代、团队扩张、渠道建设）
  <0.5 → 不战（建议：重新定位或等待）
```

### Skill 24.3: 避实击虚机会识别器

**功能**: 扫描竞品和客户痛点，找到"虚"的位置

**算法**:
```
输入: 竞品列表, 客户反馈数据, 市场增长率数据
步骤:
1. 识别"实"（竞品优势）：
   - 竞品NPS > 50, 市场份额>30%, 品牌认知度>80%
   - 标记为"实区"（避免直接竞争）
2. 识别"虚"（客户不满或空白）：
   - NPS < 30 的客户群
   - 市场增长率 > 20% 但CR3 < 50%（分散市场）
   - 客户投诉高频词（如"贵""慢""复杂"）
3. 评估"虚"的吸引力：
   虚得分 = 市场规模 × 增长倍数 × 进入壁垒倒数
   如果 > 阈值 → 推荐为"可攻击目标"
4. 输出: 3-5个"避实击虚"机会点，优先级排序
```

### Skill 24.4: 以迂为直价值评估器

**功能**: 评估"短期牺牲换取长期收益"的决策是否合理

**公式**:
```
迂直比 = 长期总价值 / 短期直接价值

长期总价值 = 终点价值 × (1 + 复合增长率)^年数 × 概率调整

判断:
  迂直比 > 3 → 强烈推荐（如平台补贴、生态建设）
  2-3 → 推荐（需确保执行质量）
  1.5-2 → 谨慎（需明确退出机制）
  <1.5 → 不推荐（性价比低）
```

**示例**:
```
选择: 开源核心产品（短期少收license费）
终点价值: $100M（如果生态建成）
年数: 5年
复合增长率: 50%（生态效应）
概率: 0.3（竞争风险）
  长期价值 = 100M × (1.5)^5 × 0.3 ≈ 22.8M
短期价值: $5M（5年license收入）
迂直比 = 22.8 / 5 = 4.6 → 推荐
```

### Skill 24.5: 修道保法组织健康评估

**功能**: 评估组织"道"与"法"的成熟度，预测战略执行力

**评估维度**:
```
修道（文化共识）:
  - 使命愿景认同度（调研，0-1）
  - 价值观践行度（行为审计，0-1）
  - 员工净推荐值(eNPS, 0-1)
  修道分 = (认同×0.4+践行×0.3+eNPS×0.3)

保法（制度流程）:
  - 决策流程明确度（0-1）
  - 绩效考核合理性（0-1）
  - 资源分配效率（ROIC vs 行业基准）
  保法分 = (决策×0.3+绩效×0.4+效率×0.3)

组织健康 = 修道×0.6 + 保法×0.4
```

**解读**:
- 健康 > 0.7 → 战略执行力强
- 0.5-0.7 → 有改进空间，需短板提升
- <0.5 → 战略无法落地，先治理再扩张
